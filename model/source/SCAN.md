# Scan
扫描一段价格走势K线，查看这段价格走势K线的尾段是否处于一个横盘价格区间内。如果在区间内需要找出价格区间的上沿和下沿并统计价格区间包含的K线数量。

## Goal
1.判断是价格走势K线的尾段是否处于一个价格横盘区间内；
2.找出价格区间的上沿和下沿；
3.统计价格区间包含的K线数量；

## Input
输入数据是某投资资产的价格走势数据。数据为四行多列，每列数据使用逗号(,)分隔。这四行数据从上到下依次为Open, High, Low, Close价格，每一列为一根K线。
数据列从左到右表示时间流逝的方向，最末尾的数据为当前最新数据。

## Output
```json
{
  "hasRange": true,
  "upper": "732.5",
  "lower": "754.5",
  "count": 98,
  "type": "bull"
}
```
输出格式说明：
hasRange字段，true表示价格走势K线的尾段处于一个横盘价格区间内，false表示没有；
upper字段，填写横盘价格区间的上沿价格数值，如果hasRange字段为false，则此处填写0；
lower字段，填写横盘价格区间的下沿价格数值，如果hasRange字段为false，则此处填写0；
count字段，填写横盘价格区间的分时K线数量，如果hasRange字段为false，则此处填写0；
type字段，填写横盘价格区间的类型：如果横盘价格区间中有至少2次触及区间价格上沿则为bull，如果横盘价格区间中有至少2次触及区间价格下沿则为bear，如果既是bull又是bear则为hybrid，如果hasRange字段为false则为none；

## Rules
1.价格区间上沿和下沿的价格差值不能大于资产价格的2.5%；
2.横盘价格区间的K线数量不能小于74根分时K线；
3.横盘价格区间中要有至少2次价格触及区间上沿(bull)，或者要有至少2次价格触及区间下沿(bear)；
4.价格触及区间上沿或者下沿并不是要绝对精确，允许有价格正负0.2%的偏差；
5.相邻的两次触及区间上沿的位置之间要至少间隔10根K线，否则只能当做一次触及区间上沿；触及区间下沿的次数统计也遵行此要求；
